code quality¶

Code quality matters : feel free to write the author to improve it !

global tests¶

We verify code’s quality with two tests :
./header_please.py verifies that each file begins with a regular header
./pylint_tests launches Pylint and gives a mark to every file in the project. Pylint’s configuration’s file has been modified this way :
* max-line-length=100
* disable=W0142 (used * or ** magic)
* max-public-methods=25
* max-args=10
* max-locals=20

unittesting¶

This program uses nosetest for unit testing; unit tests are stored in /tests/. To launch the unit tests : ./unittests.sh
