qualité du code¶

La qualité du code compte : n’hésitez pas à contacter l’auteur pour l’améliorer !

tests généraux¶

La qualité du code est vérifiée grâce à deux tests :

./header_please.py vérifie que chaque fichier a une en-tête correcte.

./pylint_tests lance Pylint qui note chaque fichier du projet. Le fichier de configuration de Pylint est modifié de la façon suivante :
* max-line-length=100
* disable=W0142 (used * or ** magic)
* max-public-methods=25
* max-args=10
* max-locals=20

unit testing¶

Ce programme utilise nosetest pour ses tests unitaires, regroupés dans /tests/. Pour lancer les tests unitaires : ./unittests.sh
